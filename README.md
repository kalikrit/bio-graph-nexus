# BioGraph Nexus

**Демонстрационное веб-приложение для построения графа знаний из текста биографии.**

Приложение принимает текст на английском языке, с помощью NLP-пайплайна (spaCy) извлекает сущности (люди, организации, места, даты) и связи между ними, сохраняет результат в графовую базу данных Neo4j и отображает интерактивный граф с возможностью получения рекомендаций.

## Ключевые возможности

- **Извлечение сущностей и связей в реальном времени** – работает на одном предложении или развёрнутой биографии
- **Граф знаний в Neo4j** – уникальность узлов гарантирована, дубликаты исключены
- **Интерактивная визуализация** – зум, панорамирование, цвета и формы по типам сущностей
- **Сайдбар с деталями** – кликните на узел, чтобы увидеть все связанные факты
- **Рекомендательная система** – поиск похожих людей по общим организациям, местам или знакомым
- **Автоматическое связывание** – места и даты привязываются к персонам через предлоги (in, at, on)
- **Готовые примеры** – встроенные тексты для мгновенной демонстрации
- **Docker-развёртывание одной командой** – Neo4j, бэкенд и фронтенд работают в изолированных контейнерах (в процессе)

## Быстрый старт (без Docker, для разработки)

### 1. Клонирование репозитория
```bash
git clone <url-вашего-репозитория>
cd bio-graph-nexus
```

### 2. Запуск Neo4j
Убедитесь, что Docker установлен, и выполните в корне проекта:
```bash
docker compose up -d neo4j
```
База данных доступна по адресу `bolt://localhost:7687` (веб-интерфейс: http://localhost:7474, логин/пароль задаются в `.env`).

### 3. Бэкенд
```bash
cd backend
poetry install                # установка всех зависимостей
poetry run spacy download en_core_web_lg  # загрузка NLP-модели
```

Создайте файл `.env` (можно скопировать из `.env.example`) и укажите параметры подключения:
```
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=ваш_пароль
SPACY_MODEL=en_core_web_lg
```

Запустите сервер:
```bash
poetry run uvicorn app.main:app --reload --reload-dir app
```
Swagger-документация API доступна по адресу http://localhost:8000/docs.

### 4. Фронтенд
В другом терминале, из корня проекта:
```bash
cd frontend
npm install
npm run dev
```
Приложение откроется на http://localhost:5173.

## Использование

1. Откройте фронтенд в браузере.
2. Выберите готовый пример (Стив Джобс, Мария Кюри, Четыре товарища) или вставьте свой текст.
3. Нажмите **«Обработать»** – через 1-2 секунды появится граф.
4. Кликайте на узлы, чтобы увидеть связанные факты в боковой панели.
5. Нажмите **«Найти похожих людей»**, чтобы получить рекомендации.

## API (основные эндпоинты)

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/extract` | Принимает `{"text": "..."}`, возвращает сущности, связи и `graph_id` |
| GET  | `/recommend?entity_id=...` | Возвращает список похожих персон с весами |

## Архитектура

```
Пользовательский интерфейс (Vue 3 + Cytoscape.js)
       │
       ▼
API Gateway (FastAPI)
       │
       ├── NLP-пайплайн (spaCy)
       │    ├── entity_extractor (сущности)
       │    └── relation_extractor (связи)
       │
       └── Репозиторий Neo4j
            ├── сохранение графа (MERGE)
            ├── получение визуализации
            └── рекомендации (Cypher)
```

## NLP-пайплайн

- **Извлечение сущностей** (`entity_extractor`): ручной обход токенов для объединения составных имён, фильтрация нужных типов (PERSON, ORG, GPE, DATE), генерация уникального `entity_id` на основе нормализованного имени и типа.
- **Извлечение связей** (`relation_extractor`):
  - **Строгие паттерны**: для ключевых глаголов (`found`, `work`, `live`, `born` и т.д.) с контролем типов субъекта и объекта.
  - **Жадный режим** (`GREEDY_MODE`): автоматически создаёт связи для любых глаголов между людьми, организациями и местами, исключая вспомогательные (`be`, `have`, `become` и т.п.).
  - **Предложные связи**: привязка мест и дат к персонам через предлоги `in`, `at`, `on`, `from`, `to`.
- **Разрешение местоимений**: простой кэш последней персоны для связывания `he`/`she` с антецедентом.
- **Нормализация**: удаление корпоративных суффиксов, приведение к единому регистру для предотвращения дубликатов в графе.

## Рекомендации

Для выбранной персоны система находит других людей, у которых больше всего общих связей (организаций, мест или других людей). Вес рекомендации равен количеству общих сущностей. Запрос выполняется одним Cypher-запросом без внешних библиотек.

## Демо-примеры

- **Стив Джобс**: `Steve Jobs founded Apple Inc. in Cupertino on April 1, 1976. He later worked at Pixar and NeXT.`
- **Мария Кюри**: `Marie Curie was born in Warsaw. She studied at the Sorbonne and discovered radium with Pierre Curie.`
- **Социальная сеть**: история четырёх друзей с браками, разводом и дружескими связями (кнопка «Четыре товарища»).
